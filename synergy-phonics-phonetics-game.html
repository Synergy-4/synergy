Game Design Specification

Phonics & Phonetics Game

A 5-stage progressive phonics curriculum that teaches children to recognise and produce sounds — from individual vowels to full sight words. Hybrid response model: tap-to-identify at beginner, microphone-based pronunciation at advanced. Pocket TTS for all audio, speech_to_text for mic recognition.

Game type: phonics_phonetics
Stages: 5 (vowels → sight words)
Response: Hybrid (receptive → expressive)
Audio: Pocket TTS with carrier phrase strategy
STT: flutter speech_to_text (word-level)
Flutter widget: PhonicsGame
// 01

5-Stage Curriculum

Every child starts at Stage 1 regardless of prior knowledge. Stages unlock sequentially — a child cannot access Stage 3 without consistent Stage 2 performance. Within each stage, items are introduced in developmental acquisition order.

S1
Vowel Sounds
Receptive Only
Phonemes
/a/ /e/ /i/ /o/ /u/
Short vowel sounds only. Each paired with an anchor image: A=apple, E=egg, I=igloo, O=orange, U=umbrella.
Mechanics
  • Hear phoneme sound → tap the correct letter from 2–3 options
  • Hear letter name + phoneme → match to anchor image
  • Introduced one at a time — A mastered before E introduced
  • No microphone at this stage
S2
Consonant Sounds
Receptive → Expressive
Acquisition Order
Group A (easiest, introduced first):
/b/ /m/ /p/ /t/ /d/ /n/
Group B (after Group A mastered):
/f/ /g/ /h/ /k/ /l/ /r/ /s/ /w/
Group C (hardest sounds):
/v/ /j/ /y/ /z/ /q/ /x/
Mechanics

  • Expressive mic mode unlocks after 3 consecutive correct receptive sessions per consonant
  • Carrier phrase TTS: "The sound is... /b/" → child repeats just "/b/"
  • Groups A, B, C unlock sequentially — never mixed until Group A mastered

Progression Gates

Two types of gates: stage gates (unlock the next stage) and mode gates (unlock expressive mic within a stage). Both are managed by the server — the client never decides progression.

Stage Unlock Gates
Stage 1 → Stage 2 unlocks when:
All 5 vowels (/a/ /e/ /i/ /o/ /u/) show ≥ 80% correct_independent rate across at least 3 sessions each. Vowels can be mastered in any order — all 5 must be consistent before any consonants are introduced.
gate: vowel_mastery_count == 5
Stage 2 → Stage 3 unlocks when:
All 6 Group A consonants (B, M, P, T, D, N) show consistent correct_independent performance. Group B and C consonants do NOT need to be mastered — the child can continue learning consonants in parallel with Stage 3.
gate: consonant_group_a_mastered == true
Stage 3 → Stage 4 unlocks when:
The 3 core digraphs (SH, CH, TH) are consistently correct receptively. Child does not need to master all blends — just the foundational digraphs.
gate: sh_correct && ch_correct && th_correct
Stage 4 → Stage 5 unlocks when:
≥ 80% accuracy across at least 5 different CVC word families. Words within a family count separately — child must demonstrate transfer across families (not just -at words).
gate: cvc_families_mastered >= 5
Stage 5 — ongoing
No upper gate. Sight words expand as the child masters batches of 5. The LLM selects from mastered-adjacent sight words each session.
gate: none — continuous
Expressive Mode Gate (within any stage)
🎙️
Mic unlocks per-phoneme, not per-stage. For each individual sound (e.g. /b/, SH, CAT), the expressive mic mode activates only when that specific sound has achieved 3 consecutive receptive correct_independent outcomes. A child may have mic enabled for /b/ but not yet for /f/ — they are gated independently. The server tracks this per-phoneme in the session history and sets response_mode: "expressive" in the SDUI config when the gate is met.

Receptive vs Expressive Mode

Receptive Mode — Tap to Identify

Applies to all stages at acquisition level. The phoneme or word plays through the speaker. The child selects the correct answer from 2–4 letter/word options displayed on screen.

Parameter Easy Medium Hard
Options shown 2 (target + 1 distractor) 3 options 4 options
Distractor type Phonemically distant (/a/ vs /o/) Same category (/a/ vs /e/) Perceptually similar (/b/ vs /d/)
Audio repeats Auto-repeats 2× before timeout Repeats once on tap of speaker icon Plays once, no repeat
Response window 10 seconds 7 seconds 5 seconds
Anchor image shown Yes — always visible Fades after 2s Not shown
Expressive Mode — Speak Into Mic

Activates only after receptive mastery of a specific phoneme. The phoneme plays, then a large mic button appears. The child holds the button and speaks. The app records and scores the attempt.

⚠️
Non-verbal children: The expressive mic button must always have a visible "Skip" option next to it. If a child skips expressive mode 3 times in a row for the same phoneme, the server should flag therapist_flag: true and include a note that expressive practice may need in-person support. Never force mic mode — it must always feel optional.
Interaction step What happens
1. Phoneme plays TTS audio plays automatically. Large phoneme letter shown on screen. "Now you try!" prompt appears.
2. Child holds mic button Waveform visualiser activates — child sees their voice as a wave. Recording begins.
3. Child releases button Recording ends. "Checking..." spinner shows briefly (max 2s).
4. STT scores attempt See scoring logic below. Result shown immediately.
5. Feedback Correct: celebration. Close: "Almost! Try again." Miss: phoneme replays, gentle encouragement.

Pocket TTS Phoneme Strategy

Isolated phonemes are the hardest case for TTS. This section defines exactly how each type of sound should be generated to maximise audio quality.

⚠️
The core problem: Asking Pocket TTS to generate "/b/" alone will produce "bee" (the letter name) or "/buh/" (with an unwanted schwa vowel). The child hears the wrong sound and learns incorrect pronunciation. The mitigation is the carrier phrase strategy below.
Carrier Phrase Strategy by Stage
Stage Phoneme Type TTS Input String Notes
S1 Vowel sounds "The letter A makes the sound... a, a, a" Repeating the phoneme 3× helps TTS produce it more cleanly than once. Server generates and caches.
S2 Consonant sounds (stops: b, p, t, d) "The sound is... buh... buh... buh" Unavoidable slight schwa on stops. Acceptable — children learn from this context. Do NOT try to generate a pure /b/ — TTS cannot do it.
S2 Consonant sounds (continuants: m, n, s, f) "The sound is... mmmm... mmmm" Continuants can be held and TTS handles them well. Generate the sustained version.
S3 Digraphs "The letters S-H make the sound... shhhh" TTS handles digraph sounds well when spelled phonetically in the input.
S3 Blends "The blend B-L sounds like... bl, bl, bl — as in blue" Include the anchor word at the end — helps TTS produce the blend in context.
S4 CVC words (whole word) "The word is... cat" Full words are TTS's strength. No carrier phrase tricks needed.
S4 CVC phoneme-by-phoneme Generate separately: "c", "a", "t" using S1/S2 strategy Client plays them in sequence with 400ms gaps between each.
S5 Sight words + full words "The word is... the" Clean TTS output. No special handling needed.
💡
All phoneme audio is pre-generated at server startup, not on-demand. The phoneme library is finite and known — 5 vowels, 21 consonants, 19 digraphs/blends. That's 45 audio files total for Stages 1–3. Generate them all at startup and cache permanently in R2/local storage. Stage 4–5 words are generated on-demand per child (themed to interests) and cached by content hash as per the TTS pipeline.

Speech Recognition — speech_to_text

The speech_to_text Flutter package transcribes what the child says. Scoring is word-level — the transcription is matched against expected values. Phoneme-level scoring is a future upgrade path.

Integration Code
Dartlib/games/phonics/stt_scorer.dart
import 'package:speech_to_text/speech_to_text.dart'; class SttScorer { final SpeechToText _stt = SpeechToText(); bool _available = false; Future<void> init() async { _available = await _stt.initialize( onError: (e) => debugPrint('STT error: $e'), ); } /// Records child's attempt and scores against [expected]. /// Returns a PhonicsScore — never throws, degrades gracefully. Future<PhonicsScore> scoreAttempt({ required String expected, // e.g. "cat" or "sh" required String stage, // "vowel" | "consonant" | "blend" | "cvc" | "word" Duration timeout = const Duration(seconds: 5), }) async { if (!_available) return PhonicsScore.unavailable(); String? transcript; await _stt.listen( onResult: (result) { if (result.finalResult) { transcript = result.recognizedWords.toLowerCase().trim(); } }, listenFor: timeout, localeId: 'en_US', cancelOnError: false, partialResults: false, ); await Future.delayed(const Duration(milliseconds: 200)); await _stt.stop(); return _evaluate(transcript, expected, stage); } PhonicsScore _evaluate(String? transcript, String expected, String stage) { if (transcript == null || transcript.isEmpty) { return PhonicsScore(result: PhonicsResult.noAudio, transcript: null); } final clean = _normalise(transcript); final exp = _normalise(expected); // 1. Exact match if (clean == exp) { return PhonicsScore(result: PhonicsResult.correct, transcript: transcript); } // 2. Transcript CONTAINS the expected word/sound // e.g. child says "buh buh" when expected is "b" — still correct if (clean.contains(exp)) { return PhonicsScore(result: PhonicsResult.correct, transcript: transcript); } // 3. For single phonemes: check accepted variants list if (stage == 'vowel' || stage == 'consonant' || stage == 'blend') { final variants = _phonemeVariants[exp] ?? []; if (variants.any((v) => clean.contains(v))) { return PhonicsScore(result: PhonicsResult.correct, transcript: transcript); } } // 4. Close match — edit distance ≤ 1 for CVC and word stages if ((stage == 'cvc' || stage == 'word') && _editDistance(clean, exp) == 1) { return PhonicsScore(result: PhonicsResult.close, transcript: transcript); } return PhonicsScore(result: PhonicsResult.miss, transcript: transcript); } String _normalise(String s) => s.toLowerCase().replaceAll(RegExp(r'[^a-z]'), '').trim(); /// Accepted STT transcriptions per phoneme /// e.g. child says "bee" when asked for /b/ — still correct static const Map<String, List<String>> _phonemeVariants = { 'b': ['bee', 'buh', 'be'], 'm': ['em', 'mmm'], 'p': ['pee', 'puh'], 't': ['tee', 'tuh'], 'd': ['dee', 'duh'], 'sh': ['shh', 'shhh', 'shush'], 'ch': ['chuh', 'church'], 'th': ['the', 'thee'], // ... full variant map in implementation }; }
Score Result Types
Result Condition Feedback Retry?
correct Exact match, contains match, or accepted variant Celebration animation + star token No — advance
close Edit distance 1 (CVC/word only) "Almost! Try one more time" — phoneme replays Yes — one retry allowed
miss No match, edit distance >1 Gentle "Let's hear it again" — replays audio, no negative signal No — move on
noAudio STT returned empty or null "Hmm, I didn't hear that — try again!" — does NOT count as miss Yes — retry free
skipped Child tapped "Skip" No feedback — move on gently. Logged for therapist flag tracking. No
unavailable STT not available (permission denied or offline) Silently fall back to receptive mode for this session. Parent notified via session summary. N/A

Scoring Logic

Pseudocodescoring per trial
// Base points receptive_correct_independent = 10 pts receptive_correct_prompted = 5 pts // answered after audio replay expressive_correct = 20 pts // double points for speaking expressive_close = 10 pts // partial credit for close attempt expressive_skipped = 0 pts // no penalty for skipping miss = 0 pts // no negative score — ever // Session completion bonus (no errors in session) perfect_session_bonus = 30 pts // Streak bonus (3 correct in a row) streak_bonus = 5 pts × streak_count // capped at 4 (20pts max) // Session result POSTed back (outcome field maps to Synergy standard) receptive_correct_independent → outcome: "correct_independent" receptive_after_replay → outcome: "correct_prompted" expressive_correct → outcome: "correct_independent" expressive_close → outcome: "correct_prompted" expressive_skipped → outcome: "no_response" miss → outcome: "incorrect" no_audio → not counted in outcome_distribution
