Parent Dashboard

Flutter mobile implementation plan for the Synergy parent-facing dashboard — covering progress tracking, child management, and client-side settings.

Platform: Flutter 3.x (iOS + Android)
State: Riverpod
Storage: SharedPreferences + SecureStorage
Charts: fl_chart
Phases: 4 · ~3 Weeks
// 01

Screen Inventory

Six screens make up the parent dashboard. All sit behind authentication, accessed via a persistent bottom navigation bar.

🏠
Home Dashboard
/dashboard

Entry point after login. Shows active child summary, today's activity prompt, streak, and quick-access to start a session.

  • Child selector chip (multi-child switcher)
  • Streak card with consecutive days
  • "Start Today's Activity" CTA card
  • Recent sessions mini-list (last 3)
  • Active goal domain pills
📈
Progress Screen
/progress

Per-skill accuracy trends for the active child. Charts sourced from GET /sessions/{child_id}/progress.

  • Domain filter tabs (All / Communication / Social / etc.)
  • Per-skill accuracy line charts (fl_chart)
  • Session count + average score summary cards
  • Goal status badges (active / mastered / paused)
  • Empty state for children with no sessions yet
👧
Children List
/children

Master list of all children under the account. Gateway to add, edit, or remove a child profile.

  • Child cards with avatar, name, age, goal count
  • Add Child FAB (floating action button)
  • Swipe-to-reveal Edit / Delete actions
  • Active child indicator badge
✏️
Add / Edit Child
/children/edit

Single form screen for creating or editing a child profile. Reused for both add and edit flows with different pre-filled state.

  • Avatar picker (emoji-based, no camera required)
  • Name field + date of birth picker
  • Interests multi-select chip input
  • Diagnosis notes text field (optional)
  • Goals section — add/remove developmental goals
⚙️
Settings
/settings

All client-side preferences. Persisted to SharedPreferences — no server calls required. Includes personal info, audio, display, and accessibility.

  • Personal info section (name, email — calls API on save)
  • Audio: music toggle, sound effects toggle, volume slider
  • Display: text size selector, high contrast toggle
  • Accessibility: voiceover speed, reduce animations
  • Account: change password, sign out, delete account
🔍
Session Detail
/sessions/:id

Drill-down from Progress or Home. Shows a single session's full result breakdown and parent notes.

  • Score breakdown (correct / prompted / incorrect)
  • Duration and completion status
  • Parent rating display
  • Parent notes (editable inline)
  • Activity name and skill domain tag

Navigation Architecture

go_router handles all routing. A persistent BottomNavigationBar with 4 tabs is the primary navigation pattern. Full-screen routes (Add/Edit Child, Session Detail) push on top.

AppRouter (go_router) │ ├── /auth → not authenticated → redirect here │ ├── /auth/login │ └── /auth/register │ └── / → ShellRoute (BottomNavBar persistent) │ ├── /dashboard Tab 0 — Home icon │ └── /sessions/:id → push (full screen, no bottom nav) │ ├── /progress Tab 1 — Chart icon │ └── /progress/skill/:skill_id → push (expanded single-skill chart) │ ├── /children Tab 2 — People icon │ ├── /children/add → push (full screen form) │ └── /children/:id/edit → push (same form, pre-filled) │ └── /settings Tab 3 — Gear icon └── /settings/change-password → push
ℹ️
ShellRoute pattern: go_router's ShellRoute wraps the 4 main tabs in a single scaffold with a persistent BottomNavigationBar. Pushing sub-routes (e.g. /children/add) goes full-screen with a back arrow and hides the bottom bar — standard mobile UX.

// 03

Home Dashboard — Feature Spec

Active Child Switcher

A horizontally scrollable row of child avatar chips at the top of the screen. Tapping a chip sets that child as active — all data on Home and Progress screens updates reactively via Riverpod.

💡
Active child is global state. Store activeChildId in a top-level Riverpod StateProvider. All providers that depend on child data (sessions, progress, goals) watch this provider and re-fetch when it changes. This single pattern eliminates prop-drilling across the entire app.
Streak Card

Shows consecutive days with at least one completed session. Computed client-side by scanning session_history dates in descending order. No server endpoint needed — derive from existing sessions data.

Dartlib/features/home/streak_calculator.dart
int calculateStreak(List<Session> sessions) { if (sessions.isEmpty) return 0; // Get unique dates with at least one completed session final completedDates = sessions .where((s) => s.completed) .map((s) => DateUtils.dateOnly(s.playedAt)) .toSet() .toList() ..sort((a, b) => b.compareTo(a)); // descending if (completedDates.isEmpty) return 0; final today = DateUtils.dateOnly(DateTime.now()); if (completedDates.first != today && completedDates.first != today.subtract(const Duration(days: 1))) { return 0; // streak broken } int streak = 1; for (int i = 1; i < completedDates.length; i++) { final diff = completedDates[i - 1].difference(completedDates[i]).inDays; if (diff == 1) streak++; else break; } return streak; }
Start Activity CTA Card

Prominent card that calls GET /activities/next/{child_id} and navigates to the game screen. Shows loading shimmer while fetching. If a session was already completed today — show a "Well done! Come back tomorrow" state instead of hiding the card.

Recent Sessions List

Last 3 sessions pulled from GET /sessions/{child_id}/history?limit=3. Each row shows: skill name, domain tag, score percentage, date. Tapping navigates to Session Detail.

Progress Screen — Feature Spec

The core feature. Shows per-skill accuracy trends so parents can see how their child is developing across all active goals.

Domain Filter Tab Bar

A horizontally scrollable TabBar with: All · Communication · Social · Cognitive · Emotional · Daily Living · Executive. Selecting a tab filters the skill chart list below. The "All" tab is always visible and selected by default.

Per-Skill Accuracy Chart Cards

One card per active skill/goal. Each card contains a LineChart from fl_chart showing accuracy (% correct_independent) over the last 10 sessions. Cards are vertically stacked in a ListView.

Dartlib/features/progress/skill_chart_card.dart
/// Data shape expected by the chart class SkillChartData { final String skillId; final String skillLabel; // e.g. "Object Identification" final String domain; // e.g. "communication" final String moduleId; // e.g. "M02" final String status; // active | mastered | paused final List<AccuracyPoint> points; // x=session index, y=accuracy 0.0–1.0 final double currentAccuracy; // most recent session final double averageAccuracy; // rolling average final int totalSessions; } class AccuracyPoint { final int sessionIndex; // x-axis final double accuracy; // y-axis: correctIndependent / total final DateTime date; }
ℹ️
Accuracy calculation: accuracy = correct_independent / total_trials. Do NOT include correct_prompted in the numerator — this is intentional. Prompted correct responses are progress, but independent responses are the mastery signal. Show both in the session detail drill-down.

Empty State

If active child has no sessions: show an illustrated empty state with message "No sessions yet — start your first activity with [name]!" and a button that navigates to Home to start an activity. If active child has sessions but none for the filtered domain: show domain-specific empty state.

Child Management — Feature Spec

Children List Screen

Fetches from GET /children. Rendered as a ListView of ChildCard widgets. No limit on number of children — the list scrolls.

ChildCardSwipe reveals edit / delete actions
Uses Flutter's Dismissible widget with secondary background (left swipe = delete confirmation, right swipe = edit). Child's avatar is an emoji displayed in a coloured circle. Card shows: avatar, name, age, number of active goals, last session date. If this is the active child: a small coloured dot indicator appears in the top-right corner of the card.
Add / Edit Child Form

Single reusable form screen. In edit mode, the form is pre-populated from the existing child object. Uses react_hook_form equivalent — TextEditingController per field with a GlobalKey<FormState> for validation.

⚠️
Diagnosis notes sensitivity. Add a visible disclaimer below the field: "This information is private and only shared with your care team." Do not show diagnosis_notes anywhere in the UI outside of the edit form — it should never appear on cards, charts, or summaries visible in public settings.
Goals Sub-Section (within Add/Edit form)

Below the child info fields, an expandable "Learning Goals" section. Parents can see existing goals and add new ones via a bottom sheet. Goals added here are POSTed to /children/{id}/goals.

AddGoalBottomSheetModal bottom sheet for adding a goal
Fields: Domain selector (segmented button or dropdown), Description text field (free text — what should the child achieve?), Priority (1–3 radio buttons: High / Medium / Low), Set By (Parent / Clinician toggle). Save button POSTs and dismisses the sheet. The goals list in the form updates optimistically.
Delete Child Flow

Swipe-left or a delete option in the edit screen triggers a confirmation AlertDialog: "Delete [name]? This will permanently remove all their sessions and progress. This cannot be undone." Two buttons: Cancel (dismisses) and "Delete [name]" (red, calls DELETE /children/{id}). On success, if deleted child was the active child, switch active child to the next available child or show empty state.

Settings Screen — Feature Spec

All preferences are client-side using SharedPreferences. Only personal info (name, email) calls the API on save. Settings take effect immediately — no "Apply" button needed.

Settings Sections
Personal InfoCalls PATCH /users/me on save
Full name field (editable inline), email (read-only, shown for reference), "Change Password" list tile navigates to dedicated screen. All changes are held in local state until "Save" is tapped — show a save button only when a change has been made (dirty state detection).
Audio SettingsClient-side only — SharedPreferences
Background Music — SwitchListTile. Key: pref_music_enabled. Default: true.

Sound Effects — SwitchListTile. Key: pref_sfx_enabled. Default: true. When disabled, all in-game tap sounds and reward sounds are silenced.

Volume — Slider (0.0–1.0). Key: pref_volume. Default: 0.8. Only visible when music or sfx is enabled. Debounce saves by 500ms to prevent excessive SharedPreferences writes on drag.
Display SettingsClient-side only — SharedPreferences
Text Size — Three-option segmented button: Small / Normal / Large. Key: pref_text_scale. Values: 0.85 / 1.0 / 1.2. Applied via a top-level MediaQuery textScaleFactor override in MaterialApp. Default: Normal.

High Contrast Mode — SwitchListTile. Key: pref_high_contrast. Default: false. When enabled, replaces the SDUI theme colors with high-contrast equivalents (dark backgrounds, white text, thick borders). Applied via a ThemeMode wrapper that the SDUI engine reads.
Accessibility SettingsClient-side only — SharedPreferences
Voiceover Speed — Slider (0.5–1.5). Key: pref_tts_rate. Default: 1.0. Passed to the TTS engine as the speech rate for all voiceover_text fields in the SDUI payload.

Reduce Animations — SwitchListTile. Key: pref_reduce_animations. Default: false. When true, replaces all Lottie celebration animations with static images and disables flutter_animate entrance effects throughout the app.
Account ActionsDanger zone — confirm before executing
Sign Out — Clears JWT tokens from flutter_secure_storage, clears all cached Riverpod state, navigates to /auth/login. No confirmation dialog needed — not destructive.

Delete Account — Shows a two-step confirmation: first dialog "Are you sure?" then a second dialog requiring the user to type "DELETE" before the button becomes active. Calls DELETE /users/me. On success: clears all local storage and navigates to login.

State Management Design

All state is managed via Riverpod. Providers are organized by feature domain and kept in lib/providers/.

Provider
Type
Purpose
activeChildId
StateProvider<String?>
Currently selected child ID. Watched by all child-dependent providers. Persisted to SharedPreferences so selection survives app restart.
authProvider
StateNotifierProvider
Auth state (user object, loading, error). Holds JWT tokens. go_router redirect listens to auth state.
childrenProvider
FutureProvider<List>
Fetches GET /children. Cached by Riverpod. Invalidated on add/edit/delete mutations.
sessionsProvider
FutureProvider.family
Keyed by childId. Fetches session history. Re-evaluates when activeChildId changes.
progressProvider
FutureProvider.family
Keyed by childId. Fetches /progress endpoint. Transforms API response into SkillChartData list.
settingsProvider
StateNotifierProvider
All client-side preferences. Loaded from SharedPreferences on app startup. Writes back on every change.
streakProvider
Provider.family
Derived from sessionsProvider. Computes streak integer. No API call — pure computation.
Dartlib/providers/settings_provider.dart
class AppSettings { final bool musicEnabled; final bool sfxEnabled; final double volume; final double textScale; // 0.85 | 1.0 | 1.2 final bool highContrast; final double ttsRate; // 0.5–1.5 final bool reduceAnimations; const AppSettings({ this.musicEnabled = true, this.sfxEnabled = true, this.volume = 0.8, this.textScale = 1.0, this.highContrast = false, this.ttsRate = 1.0, this.reduceAnimations = false, }); AppSettings copyWith({...}) => AppSettings(...); } class SettingsNotifier extends StateNotifier<AppSettings> { SettingsNotifier(this._prefs) : super(_loadFromPrefs(_prefs)); final SharedPreferences _prefs; void setMusicEnabled(bool value) { state = state.copyWith(musicEnabled: value); _prefs.setBool('pref_music_enabled', value); } void setVolume(double value) { state = state.copyWith(volume: value); _prefs.setDouble('pref_volume', value); // debounced at widget level } // ... similar setters for each preference }

💡
Settings apply globally via MaterialApp. Wrap MaterialApp with a Consumer that watches settingsProvider. Pass textScaleFactor into a MediaQuery override. Pass themeMode based on highContrast. This means every widget in the app inherits settings automatically — no prop drilling needed.

Step-by-Step Build Plan

4 phases. Each phase is independently shippable and testable. Total estimated time: 3 weeks.

Phase 1 Shell, Navigation & Settings Storage Week 1 · Days 1–5
  • Implement go_router ShellRoute with BottomNavigationBar
    Set up the 4-tab shell. Each tab renders a placeholder screen for now. The shell preserves tab scroll position using IndexedStack so switching tabs doesn't lose state.
    NavigationUI
  • Build SettingsProvider + SharedPreferences storage layer
    Implement AppSettings model, SettingsNotifier, and SettingsStorage wrapper. Load from SharedPreferences at app startup using ProviderScope overrides. Write on every state change.
    StateStorage
  • Wire settings into MaterialApp (text scale, high contrast, animations)
    Wrap MaterialApp with a Consumer watching settingsProvider. Apply textScaleFactor via MediaQuery override, reduceAnimations via a global InheritedWidget that game widgets read.
    StateUI
  • Build the full Settings screen — all 5 sections
    Audio (music toggle, SFX toggle, volume slider), Display (text size segmented button, high contrast), Accessibility (TTS rate slider, reduce animations), Personal Info (name/email fields), Account Actions (sign out, delete account dialogs). All settings take effect immediately.
    UIStorage
  • Implement ActiveChildProvider and child switcher chip row
    Global StateProvider<String?> for active child ID. Persisted to SharedPreferences. Build the horizontal chip row widget used on Home and Progress. When a new child is selected, all FutureProvider.family providers depending on childId auto-invalidate.
    StateStorage
